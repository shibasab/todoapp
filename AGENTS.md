# AGENTS.md

## 目的

本ファイルは、論文「Evaluating AGENTS.md: Are Repository-Level Context Files Helpful for Coding Agents? (arXiv:2602.11988)」の知見に基づき、**タスク成功率に寄与する最小限の指示だけ**を記載する。

## タスク成功率を高める運用ルール（最小要件）

1. **指示は最小限に保つ**
   - タスク達成に直接必要な要件のみ記載する。
   - 目的に無関係な禁止事項・手順・重複ルールは追加しない。

2. **タスクに関連する範囲だけ指定する**
   - 対象ディレクトリ・対象レイヤー・対象仕様を明示する。
   - タスクに無関係な全体規約を毎回強制しない。

3. **曖昧さを避け、実行可能な情報に限定する**
   - 参照先は具体パスで示す（例: `docs/specs/...`）。
   - 実行コマンドは必要なものだけを列挙する。

4. **要件の優先順位を明確にする**
   - 優先度は「ユーザー指示 > システム/開発者指示 > 本ファイル」とする。
   - 矛盾時は、低優先度ルールを適用しない。

5. **不要な要求で作業を重くしない**
   - 成果に直結しない追加作業（過剰な横断調査、不要な再整形、無関係な検証）を要求しない。

## このリポジトリで必要な最小コンテキスト

- ディレクトリ構成:
  - `backend/`: バックエンド
  - `frontend/`: フロントエンド
  - `shared/`: 共通型・ユーティリティ
  - `docs/`: 仕様・NFRドキュメント
- 実装前に対象仕様を確認:
  - 機能仕様: `docs/specs/NNN_feature/spec.md`
  - 非機能要件: `docs/nfr/NNN_topic/nfr.md`
- 仕様に未記載の内容は未決事項としてユーザー確認する。

## コミュニケーション

- ユーザー向けの説明・コメント・PR本文は日本語で記述する。
